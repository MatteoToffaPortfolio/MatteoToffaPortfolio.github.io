01 — Process Automation

Automazione DDT

Pipeline Python per il parsing automatico di Documenti di Trasporto in formato PDF: estrazione dati, rilevamento incongruenze e generazione di output strutturati in Excel — eliminando l'inserimento manuale e riducendo errori operativi.

Python Pandas PDF Parsing Excel Process Automation Data Validation

Obiettivo del progetto

Automatizzare il processo di lettura e controllo dei Documenti di Trasporto (DDT) aziendali. Il flusso originale richiedeva inserimento manuale dei dati da PDF a Excel, con alto rischio di errori e tempi di elaborazione elevati. La pipeline sviluppata elimina questo collo di bottiglia operativo.

Pipeline di automazione

01

Input — PDF DDT

Lettura batch di file PDF dei documenti di trasporto dalla cartella di input.

02

Parsing & Estrazione

Estrazione strutturata dei campi chiave: codici articolo, quantità, prezzi, date, mittente e destinatario.

03

Validazione & Controllo incongruenze

Verifica automatica di coerenza tra i campi (prezzi, quantità, codici). Flagging delle righe con anomalie per revisione manuale.

04

Output — Excel strutturato

Generazione automatica del file Excel con dati puliti, fogli separati per DDT validi e righe con incongruenze.

Risultati

Esempio di output — dati estratti e strutturati in Excel
Report incongruenze — righe flaggate per revisione
Pipeline di elaborazione batch

Impatto

  • Eliminazione dell'inserimento manuale dei dati da PDF
  • Riduzione del rischio di errori operativi tramite validazione automatica
  • Elaborazione batch di più DDT in pochi secondi
  • Output standardizzato pronto per analisi e reportistica
